# Wedding Butler V6.8 — 個人行程重複資訊合併版

- 已連結婚禮流程的工作，會合併到同一張婚禮流程卡片。
- 合併卡片時間會涵蓋工作開始時間與流程時間，例如 10:20－10:30。
- 工作細項、備註與地點仍會保留在流程卡片內。
- 未連結婚禮流程的工作仍獨立顯示。
- 不以名稱硬合併，而是依既有 flowIds 關聯判斷。
- 不需調整 Firestore collections 或 rules。

# Wedding Butler V6.4｜個人行程名單同步版

本版以 V6.3 為基礎，調整「我的行程」資料來源：

- 婚禮流程負責人：顯示完整流程。
- 流程確認項負責人：顯示本人負責的小項。
- 流程掛載的工作名單：名單內每位成員都會看到該流程，並標示所屬工作名單。
- 工作負責人：顯示本人工作。
- 準備物品：不再顯示於「我的行程」，仍保留在「婚禮準備」。
- 同一流程同時符合多個身份時，依「流程負責人 → 小項負責人 → 工作名單成員」去除重複。

沿用既有 Firestore collections 與欄位，不需要修改 Firestore Rules。

## V6.5 修正
- 流程確認項名稱可直接點擊開啟修改視窗。
- 每筆確認項固定顯示「✏️ 修改」與「刪除」按鈕。
- 手機版確認項操作按鈕會自動換行，不會被擠到畫面外。
- 個人行程保留垂直線貫穿效果，並提升線與節點圖層，避免被卡片遮住。

## V6.6 個人行程時間線三欄重構
- 個人行程固定分成「時間、貫穿線、行程卡片」三欄。
- 垂直線保留連續穿過的時間軸效果。
- 卡片限制在右側欄位，不會再覆蓋或切斷時間線。
- 圓點與卡片間新增短連接線，桌機與手機皆會自動調整欄寬。


## V6.8.1 修正
- 修復「加確認項」按鈕無反應。
- 修復確認項只能刪除、無法修改。
- 恢復確認項備註欄位。
- 加入防呆，避免欄位缺失時整個編輯視窗中斷。
