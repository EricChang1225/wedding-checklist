# Wedding Control Center v14：三層收合版

## 婚禮流程現在支援三層收合

### 第一層：婚禮階段
例如：
- 文定
- 迎娶
- 婚宴

整個階段可以收合，平常只看各階段的進度。

### 第二層：流程事件
例如：
- 07:30 集合
- 08:00 文定物品清點
- 11:00 迎娶集合

每個流程都可以獨立收合。

### 第三層：準備包
例如：
- 女方準備
- 男方準備
- 額外準備

每個準備包都可以收合，展開後才顯示金飾、喜餅、聘金等細項。

每個流程另外提供「全部展開／收合」，可一次控制該流程內所有準備包。

## 顯示方式
即使收合，仍會看到：
- 完成數量
- 進度條
- 負責人
- 地點與 Google 地圖

## 資料
原有流程、階段、準備項目與細項全部保留，不需要重建。

## Firebase
沒有新增 collection，也沒有修改規則需求，不需要重新發布 firestore.rules。
